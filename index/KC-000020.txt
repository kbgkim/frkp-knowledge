JOB ID 를 클릭하면 해당 J OB 에 대한 JOB 모니터링 화면으로 넘어가게 된다. 재 요청 한번 REQ 모니터링에 REQ ID 가 생성되었던 요청 건에 대해서는 몇 번이든 요청하여 평가할 수 있다. 재 요청 버튼을 누르면 JOB ID 가 새롭게 생성되고 최종 수정자 ID 와 최종 수정일시 란에 재 요청자의 ID 와 수정일시가 표시된다. 등록일자 & 기준일자 등록일자를 변경하여 과거 요청되었던 이력을 확인 할 수 있다. Default 값은 오늘 날짜이다. 기준일자를 변경하여 조회를 하게 되면 등록일자에 상관없이 기준일에 대해서 요청되었던 이력을 확인할 수 있다. J OB 모니터링 개요 J OB 모니터링은 고객 사에서 보유한 대상종목들의 평가작업의 상태 및 현황을 조회하는 기능을 수행한다. 화면 상세 설명 및 레이아웃 화면경로 [시장리스크 ]-[OTC]-[JOB 모니터링 ] 화면 레이아웃 화면기능 상세설명 J OB 상태 조회 J OB ID 별로 [ W ] 대기중, [ P ] 처리중, [E] 오류, [ T ] 완료 상태를 조회 할 수 있다. 요청 ID 가 생성디었을 시 JOB ID 는 2 건씩 생성되기 때문에 평가 서버가 평가작업을 하고 있는 JOB ID 에 대해서는 [ P ] 처리중 상태가 되고, 2 건 중 평가 서버가 평가 작업을 하고 있지 않는 JOB ID 는 [ W ] 대기중 상태가 된다. 평가를 완료하게 되면 평가 오류가 존재 유무에 따라 JOB 상태가 달라지게 되는데 평가 오류가 존재하지 않을 경우, [ T ] 완료 상태가 되고 평가 오류가 한 종목이라도 발생하게 되면 [ E ] 오류 상태가 된다. T ASK 상태 평가서버에 의해 평가되고 있는 종목들의 처리 상태를 나타낸다. 현재 평가 서버가 평가작업을 하고 있는 종목들에 대해서는 [ P ] 처리중으로 표시되고 여러가지 이유로 인하여 [ E ] 오류로 분류된 종목에 대해서는 오류코드와 오류 메시지를 출력하게 되며 오류 코드 체계는 KIS ERROR CODE 를 사용한다 ( 운영자 매뉴얼 참고 ). 그리고 아직 평가되지 않은 상품에 대해서는 [ W ] 대기중으로 표시된다. 아래 그림은 TASK 상태를 [ T ] 완료 로 설정하고 조회했을 경우를 보여준다. 처리 시작일시, 처리종료일시, 처리요소시간 J OB 모니터링 화면에서는 평가 처리시작일시, 평가 처리종료일시, 처리요소시간이 표시된다. 각 JOB ID 별로 평가처리시간을 확인 할 수 있는데 유의해야 할 점은 위에서 조금 언급한 바와 같이 JOB ID 는 2 건 씩 생성되기 때문에 먼저 평가작업을 하던 평가 서버가 더 이상 동일한 JOB ID 에 대해 평가해야 할 종목이 없을 경우 대기 중이던 JOB ID 의 종목을 평가 시작하기 때문에 온전히 평가서버의 모든 CORE 를 사용하지 못한다. 등록일자 & 기준일자 등록일자를 변경하여 과거 평가되었던 이력을 할 수 있다.
